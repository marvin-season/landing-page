# 代码变更历史 (.history)

本目录用于记录 AI 或开发者对项目代码的修改历史，供后续会话参考，以提升开发连贯性和响应效果。

## 目录结构

- **日期子文件夹**：以 `YYYYMMDD` 为名，例如 `20260225` 表示 2026-02-25。
- **单条变更**：当日每次变更对应一个文件，文件名为 `[总结标题].md`，例如 `新增-chat-stream-状态-hook.md`。

示例：

```
.history/
  20260225/
    新增-chat-stream-状态-hook.md
    修复-agui-页面报错.md
  20260224/
    重构-stream-状态管理.md
```

## 用途

- **修改后**：每次完成代码修改后，在当日日期文件夹下新建 `[总结标题].md`，写入本次变更摘要。
- **修改前**：若不在同一会话上下文中，先读取 `.history/` 下近期日期文件夹中的 `.md` 文件，再开始改代码。

详细约定见 `.cursor/rules/code-change-history.mdc`。
