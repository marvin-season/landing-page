# 代码变更历史（.changelog）

本目录用于记录 AI 或开发者对项目代码的修改历史，供后续会话参考，以提升开发连贯性和减少冲突。

## 目录与文件规则

- **日期子文件夹**：以当天日期为子文件夹名，格式为 `YYYYMMDD`，例如 `20260225` 表示 2026-02-25。
- **单文件规则**：当日所有变更**固定使用同一文件名** `index.md`，即 `.changelog/YYYYMMDD/index.md`。不按会话或主题拆分多个文件。
- **文件内容结构**：
  - **一级标题（`#`）**：根据当前文件内已记录的全部变更**动态总结**当日变化，用简短中文或英文概括。若为新建文件，可先写本次变更的概括；后续追加变更后可更新该一级标题以覆盖全天内容。
  - **每次修改**：在文件中**追加一条二级标题（`##`）**，标题为本次变更的简短概括和时间戳；该 `##` 下写明本条记录（建议含 **文件**、**修改内容**、**原因/上下文**）。

目录结构示例：

```
.changelog/
  20260225/
    index.md
  20260224/
    index.md
```

## 修改代码之后

完成任意代码修改后，**必须**在 `.changelog/YYYYMMDD/index.md` 中记录：

1. 若当日尚无 `index.md`：新建该文件，先写一级标题，再写第一条 `##` 记录。
2. 若当日已有 `index.md`：在文件末尾**追加**一条 `##`；可选更新顶部一级标题以概括当日全部变更。

## 修改代码之前（跨会话）

新会话、无「刚才改了什么」的上下文时，**先读取**近期 `.changelog/YYYYMMDD/index.md`（优先当天，再按日期倒序），再开始改代码。

## 维护建议

- 按日期分子文件夹查阅；旧记录可保留或按需归档。
- 是否纳入版本控制：不要将 `.changelog/` 加入 `.gitignore` 若希望随仓库版本管理。

## 条目示例

```markdown
# 规则与 agui 页优化（当日变更摘要）

## 重命名 .history 为 .changelog
- **文件**: `.changelog/README.md`
- **修改内容**: …
- **原因/上下文**: …
```

以上为**项目约定**（非 Cursor Project Rule）。是否写入 `.changelog` 由团队与你在对话中的要求决定；Cursor 自带的记忆/上下文不会替代仓库内可追溯的按日记录。
