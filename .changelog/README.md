# 代码变更历史 (.changelog)

本目录用于记录 AI 或开发者对项目代码的修改历史，供后续会话参考，以提升开发连贯性和响应效果。

## 目录结构

- **日期子文件夹**：以 `YYYYMMDD` 为名，例如 `20260225` 表示 2026-02-25。
- **单文件规则**：每个会话对应一个文件，文件名为 `[会话需求主题].md`；同一会话内的多次修改合并到该文件中，文件内保留多条需求/变更记录。

示例：

```
.changelog/
  20260225/
    主题色与用户模块.md
    agui-流式对话页优化.md
  20260224/
    重构-stream-状态管理.md
```

## 用途

- **修改后**：在当日日期文件夹下新增或追加到当前会话对应的 `[会话需求主题].md`，写入本次变更摘要。
- **修改前**：若不在同一会话上下文中，先读取 `.changelog/` 下近期日期文件夹中的 `.md` 文件，再开始改代码。

详细约定见 `.cursor/rules/code-change-history.mdc`。
